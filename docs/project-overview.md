# Project Overview: ICORF Odoo System

## Executive Summary
Sistem ICORF (Internal Controls Over Financial Reporting) ini adalah modul Odoo 19.0 yang dirancang untuk memastikan kepatuhan 100% terhadap **POJK No. 15 Tahun 2024** dan **SK-5/DKU.MBU/11/2024** (Kementerian BUMN). Sistem ini mengikuti kerangka kerja **COSO 2013** dan **COBIT 2019** dengan menerapkan model **Three Lines of Defense**.

## Core Purpose
Tujuan utama sistem ini adalah menyediakan platform terintegrasi untuk:
1.  **Dokumentasi Kontrol:** Mengelola Risk Control Matrix (RCM) yang komprehensif.
2.  **Scoping Otomatis:** Menentukan akun dan lokasi signifikan menggunakan Aturan 2/3 (Cakupan >= 66.7%).
3.  **Pengujian Audit:** Mendukung Test of Design (TOD) dan Test of Operating Effectiveness (TOE).
4.  **Remediasi & Monitoring:** Melacak perbaikan defisiensi dengan validasi masa tunggu (remediation lock).
5.  **Pelaporan Kepatuhan:** Menghasilkan laporan formal seperti Laporan DOD, RCM, dan Statement Letter CEO/CFO.

## Tech Stack Summary
- **Framework:** Odoo 19.0 (Python, PostgreSQL)
- **Architecture:** Odoo MVC (Model-View-Controller)
- **Primary Modules:** `icofr`, `icofr_demo`, `icofr_demo_new_features`
- **Dependencies:** `base`, `account`, `mail`, `web`

## Key Compliance Features
- **Visual Risk Heatmap:** Matriks 5x5 untuk analisis kemungkinan dan dampak.
- **Cooling-off Period:** Validasi 12 bulan untuk independensi auditor.
- **Sampling Calculator:** Perhitungan jumlah sampel otomatis berdasarkan frekuensi dan tingkat risiko (Tabel 22).
- **Audit Dossier Wizard:** Pengumpulan bukti audit satu-klik.
