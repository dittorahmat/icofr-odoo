# Architecture Documentation: ICORF Odoo System

## 1. Architectural Pattern
Sistem ini menggunakan pola **Odoo MVC (Model-View-Controller)** yang dimodifikasi untuk mendukung tata kelola **Three Lines of Defense**:
- **Model:** Python classes di `icofr/models/` yang mengelola status dan validasi kepatuhan.
- **View:** XML files di `icofr/views/` yang mendefinisikan UI dan Dashboard.
- **Security:** Matriks akses di `icofr/security/` yang memisahkan wewenang antar lini pertahanan.

## 2. Data Architecture (Core Models)

### A. Compliance Framework
- `icofr.coso.element`: Pemetaan ke 17 prinsip COSO 2013.
- `icofr.cobit.element`: Pemetaan ke framework ITGC COBIT 2019.
- `icofr.industry.cluster`: Katalog risiko spesifik industri (Energi, Pangan, dll).

### B. Risk & Control (RCM)
- `icofr.risk`: Mengelola profil risiko dengan hitungan `risk_level` otomatis (Max(Quant, Qual)).
- `icofr.control`: Master data kontrol dengan validasi khusus untuk **EUC**, **IPE**, **MRC**, dan **ITGC**. Mendukung otomatisasi pengecekan efektivitas ITGC pendukung.

### C. Scoping Engine
- `icofr.materiality`: Kalkulator materialitas (OM, PM, SAD) dan validasi **Aturan 2/3** (Cakupan 66.7% metrik finansial).
- `icofr.account.mapping`: Pemetaan saldo GL ke FSLI dengan indikator signifikansi kuantitatif/kualitatif.

### D. Testing & Audit Trail
- `icofr.testing`: Alur pengujian TOD/TOE dengan kalkulator sampling (Tabel 22), validasi masa tunggu remediasi (Tabel 23), dan pengecekan cooling-off auditor (Hal 19).
- `icofr.audit.population`: Penyimpanan data populasi untuk pemilihan sampel yang objektif.

## 3. Security & Access Control
Sistem menerapkan **Role-Based Access Control (RBAC)** yang ketat:
- **Line 1 (Business Unit):** Owner kontrol, pelaksana CSA, dan remediation owner.
- **Line 2 (ICOFR Team):** Pemilik metodologi, verifikator desain, dan pengelola scoping.
- **Line 3 (Internal Audit):** Pelaksana pengujian TOE independen.
- **External Auditor:** Akses Read-Only untuk verifikasi data akhir tahun.

## 4. Workflow & Automation
- **Approval Workflow:** Kontrol dan Pengujian memiliki siklus hidup `draft` -> `under_review` -> `active`.
- **Automated Scoping:** Tombol "Identify Significant Locations" di Materialitas yang melakukan evaluasi otomatis berdasarkan PM.
- **Notification Engine:** Pengiriman email otomatis untuk jadwal pengujian yang mendekati jatuh tempo.

## 5. Reporting Engine
Menggunakan **QWeb Reporting** Odoo untuk menghasilkan dokumen PDF yang siap diaudit:
- **RCM Report:** Daftar Risiko dan Kontrol terpetakan.
- **DOD (Description of Deficiency):** Rekapitulasi temuan dan dampak ke laporan keuangan.
- **Statement Letter:** Surat pernyataan efektivitas dari CEO/CFO.
