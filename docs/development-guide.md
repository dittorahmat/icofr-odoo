# Development Guide: ICORF Odoo System

Dokumen ini menjelaskan cara menyiapkan lingkungan pengembangan dan standar koding untuk modul ICORF.

## 🛠️ Environment Setup

### Prerequisites
- **Python 3.10+**
- **PostgreSQL 15+**
- **Docker & Docker Compose**
- **Odoo 19.0 Community Edition**

### Installation Steps
1.  **Clone Repository:**
    ```bash
    git clone https://github.com/your-org/icofr-odoo.git
    cd icofr-odoo
    ```
2.  **Start Services with Docker:**
    ```bash
    docker-compose up -d
    ```
3.  **Install/Update Modules:**
    Sistem dikonfigurasi untuk memuat modul secara otomatis saat startup:
    ```bash
    docker-compose exec odoo odoo -u icofr,icofr_demo -d odoo --stop-after-init
    ```

## 🧪 Testing & Validation

### Running Unit Tests
Gunakan perintah berikut untuk menjalankan unit test khusus modul `icofr`:
```bash
docker-compose exec odoo odoo -i icofr --test-enable --stop-after-init -d odoo
```

### Data Validation
- Gunakan **Template Laporan** (FSLI) untuk inisialisasi awal.
- Gunakan **General Ledger** untuk memvalidasi logika Aturan 2/3.

## 📏 Coding Standards
1.  **Strict Compliance:** Perubahan pada model `icofr.control` atau `icofr.testing` wajib mengikuti referensi Juknis BUMN yang tercantum dalam komentar kode.
2.  **Naming Convention:** Semua model baru harus diawali dengan prefix `icofr.` (misal: `icofr.new_feature`).
3.  **Security:** Selalu tambahkan baris akses baru di `security/ir.model.access.csv` untuk keempat grup utama (Lini 1, 2, 3, dan Auditor Eksternal).
