# Deployment Guide: ICORF Odoo System

Panduan ini menjelaskan konfigurasi infrastruktur dan prosedur deployment untuk sistem ICORF.

## 🏗️ Infrastructure Architecture
Sistem dideploy menggunakan arsitektur containerized berbasis Docker Compose:
- **Web App:** Odoo 19.0 (Custom Build)
- **Database:** PostgreSQL 15

## 📦 Docker Configuration

### Dockerfile
Dockerfile menginstal dependensi Python tambahan dan memetakan folder addons ke dalam container Odoo.

### Docker Compose
File `docker-compose.yml` menyediakan setup siap pakai:
```yaml
services:
  db:
    image: postgres:15
    # Data persistency di volume odoo_db_data
  odoo:
    build: .
    ports: ["8069:8069"]
    volumes:
      - ./addons:/mnt/extra-addons
      - ./odoo.conf:/etc/odoo/odoo.conf
```

## 🚀 Deployment Steps

1.  **Preparation:**
    Pastikan file `odoo.conf` telah disesuaikan dengan kredibilitas database produksi.
2.  **Pull & Build:**
    ```bash
    docker-compose build --pull
    ```
3.  **Launch:**
    ```bash
    docker-compose up -d
    ```
4.  **Database Migration:**
    Saat melakukan upgrade modul di produksi:
    ```bash
    docker-compose exec odoo odoo -u icofr -d odoo --stop-after-init
    ```

## 🛡️ Backup & Security
- **Database Backup:** Lakukan dump PostgreSQL secara rutin dari volume `odoo_db_data`.
- **Environment Variables:** Jangan simpan password DB dalam bentuk teks polos di `docker-compose.yml` untuk lingkungan produksi; gunakan file `.env`.
